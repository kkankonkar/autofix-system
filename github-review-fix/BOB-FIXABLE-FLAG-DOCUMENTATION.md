# Bob Fixable Flag Documentation

## Overview
The `bob_fixable` flag is a boolean field in the review findings schema that indicates whether a specific code review finding can be safely targeted by the Fixium implement workflow. This enables automated code fixes for specific issues identified during code review.

## Schema Definition

### Location
`schema/review_output.schema.json` (lines 106-110)

```json
"bob_fixable": {
  "type": "boolean",
  "description": "Whether this finding can be safely targeted by the Fixium implement workflow",
  "default": false
}
```

## Complete Workflow

### 1. Review Phase (`Fixium:review`)
When a user comments `Fixium:review` on a PR:

1. **Bob Shell performs code review** and generates findings in JSON format
2. **Each finding can include `bob_fixable: true`** if it's suitable for automated fixing
3. **Findings are submitted as PR review comments** via `submit_pr_comments.sh`
4. **Comment formatter adds the flag** to the comment body as `` `bob_fixable: true` ``

Example finding in JSON:
```json
{
  "file": "src/app.py",
  "line": 42,
  "severity": "medium",
  "type": "maintainability",
  "title": "Magic number should be constant",
  "description": "The magic number 42 should be extracted to a named constant",
  "suggestion": "MAX_RETRIES = 42",
  "bob_fixable": true
}
```

### 2. Comment Formatting
The `lib/comment_formatter.sh` script formats findings into GitHub markdown comments:

- **Extracts `bob_fixable` field** from JSON (line 93)
- **Adds flag to comment footer** if `bob_fixable: true` (lines 117-120)
- **Format**: `` `bob_fixable: true` `` appears at the end of the comment

Example formatted comment:
```markdown
🟡 **MEDIUM SEVERITY** | 🔧 **MAINTAINABILITY**

**Issue:** The magic number 42 should be extracted to a named constant

**Details:** Magic numbers reduce code readability

**Suggestion:** MAX_RETRIES = 42

`bob_fixable: true`

---
*🤖 Generated by Fixium Code Review | Line 42*
```

### 3. Implement Phase (`Fixium:implement`)
When a user replies to a review comment with `Fixium:implement`:

1. **GitHub workflow triggers** (`.github/workflows/fixium.yml`)
2. **Python validates the target** (`fixium/main.py:validate_implement_target()`)
   - Retrieves the review comment using `github_client.get_review_comment()`
   - Checks if comment contains `` `bob_fixable: true` `` (line 55)
   - Raises error if flag is missing
3. **Bob Shell implements the fix** based on the validated finding
4. **Changes are committed** to the PR branch

## Implementation Details

### Python Code (`fixium/main.py`)

```python
def is_bob_fixable_comment(review_comment: dict[str, Any]) -> bool:
    """Check whether a review comment is explicitly marked bob-fixable."""
    return '`bob_fixable: true`' in review_comment.get('body', '').lower()

def validate_implement_target(
    github_client: GitHubClient,
    reply_to_comment_id: Optional[int],
) -> dict[str, Any]:
    """Validate that implement command is replying to a bob-fixable review comment."""
    if reply_to_comment_id is None:
        raise ValueError("REPLY_TO_COMMENT_ID is required for Fixium:implement")
    
    review_comment = github_client.get_review_comment(reply_to_comment_id)
    
    if not is_bob_fixable_comment(review_comment):
        raise ValueError(
            f"Review comment {reply_to_comment_id} is not marked as bob_fixable"
        )
    
    return review_comment
```

### Shell Script (`lib/comment_formatter.sh`)

```bash
# Extract bob_fixable flag from JSON
local bob_fixable=$(echo "$finding_json" | jq -r '.bob_fixable // false')

# Build bob_fixable footer if applicable
local bob_fixable_footer=""
if [ "$bob_fixable" = "true" ]; then
    bob_fixable_footer=$'\n\n`bob_fixable: true`'
fi

# Include in formatted comment
cat <<EOF
${severity_badge} **${severity_upper} SEVERITY** | ${type_icon} **${type_upper}**

**Issue:** ${description}

**Details:** ${explanation}

**Suggestion:** ${suggestion}
${bob_fixable_footer}

---
*🤖 Generated by Fixium Code Review | Line ${line}*
EOF
```

## Usage Guidelines

### When to Set `bob_fixable: true`

✅ **Safe to automate:**
- Simple refactoring (extract constant, rename variable)
- Code style fixes (formatting, naming conventions)
- Adding missing documentation
- Simple bug fixes with clear solutions
- Adding error handling in straightforward cases

❌ **Not safe to automate:**
- Complex logic changes
- Security-critical modifications
- Changes requiring business logic understanding
- Fixes that need human judgment
- Changes affecting multiple files in complex ways

## Error Handling

If a user tries to implement a finding without the `bob_fixable` flag:

```
ValueError: Review comment 12345 is not marked as bob_fixable
```

This prevents accidental automated fixes on findings that require human review.
