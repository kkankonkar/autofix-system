# Fixium GitHub Actions Integration - Implementation Plan

## Overview
Transform the code-review-workflow into **Fixium** - a GitHub Actions-powered code review bot that triggers on PR comments.

## Architecture Decision: Hybrid Python + Shell

### Why Hybrid?
- **Python**: Better for API interactions, JSON processing, logic, and testing
- **Shell**: Keep for Bob Shell invocation (already works well) and git operations
- **Best of Both**: Leverage strengths of each language

### Language Breakdown

**Python Components** (New):
- Comment parsing and validation
- GitHub API client (using PyGithub)
- Access control and authorization
- Progress tracking and updates
- Error handling and messaging
- JSON processing and filtering
- Unit testing with pytest

**Shell Components** (Keep):
- Bob Shell invocation (`bob -p "..."`)
- Git operations (checkout, diff, etc.)
- File system operations
- Existing `review_workflow.sh` wrapper

## Requirements Summary

### Trigger
Comment on PR with `Fixium:review [options]`

### Supported Filter Options
- `--severity high,medium,low` - Filter by severity levels
- `--type bug,security,maintainability,performance` - Filter by issue types
- `--exclude-severity low` - Exclude specific severities
- `--exclude-type maintainability` - Exclude specific types

### Command Examples
```
Fixium:review
Fixium:review --severity high
Fixium:review --type bug,security
Fixium:review --severity high,medium --exclude-type maintainability
Fixium:review --type bug,security --exclude-severity low
```

### Workflow Behavior
1. **Parse** comment for command and filter options
2. **Authorize** - Check if user is in authorized list
3. **Concurrency Check** - Block if review already in progress
4. **Progress Start** - Post "🔍 Fixium review started..." comment
5. **Execute Review** - Run review on PR files with filters
6. **Progress Updates** - Update comment with current status
7. **Post Findings** - Submit inline comments on code
8. **Finalize** - Update comment with summary and statistics
9. **Error Handling** - Post helpful error messages if failures occur

### Access Control
- Only specific users/teams can trigger reviews
- Configured via `FIXIUM_AUTHORIZED_USERS` secret
- Unauthorized users get friendly denial message

### Concurrency Control
- Block new reviews while one is running on same PR
- Post "⏳ Review already in progress, please wait" message
- Check via GitHub Actions API for running workflows

### Branding
- Replace all "Bob Shell" references with "Fixium"
- Update comment footers: "🤖 Generated by Fixium Code Review"
- Keep internal `bob` CLI command as-is (implementation detail)

### Infrastructure
- Docker container with bob CLI + all dependencies
- GitHub Actions workflow triggered by PR comments
- Secrets: `BOBSHELL_API_KEY`, `GITHUB_TOKEN`, `FIXIUM_AUTHORIZED_USERS`

### Task 4: Comment Parser Script

**Location**: `lib/parse_comment.sh`

**Purpose**: Extract command and filter options from PR comment

**Input**: Comment body text via environment variable

**Output**: JSON with parsed options

**Implementation**:
```bash
#!/bin/bash

parse_fixium_comment() {
    local comment_body="$1"
    
    # Extract command (e.g., "review")
    local command=$(echo "$comment_body" | grep -oP 'Fixium:\K\w+')
    
    # Extract filter options
    local severity=$(echo "$comment_body" | grep -oP '\-\-severity\s+\K[^\s]+' || echo "")
    local type=$(echo "$comment_body" | grep -oP '\-\-type\s+\K[^\s]+' || echo "")
    local exclude_severity=$(echo "$comment_body" | grep -oP '\-\-exclude-severity\s+\K[^\s]+' || echo "")
    local exclude_type=$(echo "$comment_body" | grep -oP '\-\-exclude-type\s+\K[^\s]+' || echo "")
    
    # Output as JSON
    cat <<EOF
{
  "command": "$command",
  "severity": "$severity",
  "type": "$type",
  "exclude_severity": "$exclude_severity",
  "exclude_type": "$exclude_type"
}
EOF
}

# Usage: parse_fixium_comment "$COMMENT_BODY"
```

### Task 5: Access Control Script

**Location**: `lib/check_access.sh`

**Purpose**: Verify user authorization to trigger Fixium

**Input**: 
- GitHub username (from comment author)
- Authorized users list (from secret)

**Output**: Exit code 0 (authorized) or 1 (denied)

**Implementation**:
```bash
#!/bin/bash

check_user_access() {
    local user="$1"
    local authorized_users="$2"
    
    # Convert comma-separated list to array
    IFS=',' read -ra USERS <<< "$authorized_users"
    
    # Check if user is in authorized list
    for authorized_user in "${USERS[@]}"; do
        if [ "$user" = "$authorized_user" ]; then
            echo "✅ User $user is authorized"
            return 0
        fi
    done
    
    echo "❌ User $user is not authorized to trigger Fixium"
    return 1
}

post_unauthorized_message() {
    local pr_number="$1"
    local user="$2"
    
    local message="@$user Sorry, you are not authorized to trigger Fixium reviews. Please contact a repository administrator."
    
    # Post comment using GitHub API
    gh api \
        --method POST \
        -H "Accept: application/vnd.github+json" \
        "/repos/$GITHUB_REPOSITORY/issues/$pr_number/comments" \
        -f body="$message"
}
```

**Usage in Workflow**:
```bash
if ! check_user_access "$COMMENT_USER" "$AUTHORIZED_USERS"; then
    post_unauthorized_message "$PR_NUMBER" "$COMMENT_USER"
    exit 1
fi
```

### Task 6: Concurrency Lock Script

**Location**: `lib/concurrency_lock.sh`

**Purpose**: Prevent concurrent reviews on same PR

**Mechanism**: Check GitHub Actions API for running workflows

**Implementation**:
```bash
#!/bin/bash

check_concurrent_review() {
    local pr_number="$1"
    local current_run_id="$2"
    
    # Get all running workflows for this repository
    local running_workflows=$(gh api \
        -H "Accept: application/vnd.github+json" \
        "/repos/$GITHUB_REPOSITORY/actions/runs?status=in_progress" \
        | jq -r '.workflow_runs[] | select(.name == "Fixium Code Review") | .id')
    
    # Check if any other workflow is running for this PR
    for run_id in $running_workflows; do
        if [ "$run_id" != "$current_run_id" ]; then
            # Check if this run is for the same PR
            local run_pr=$(gh api \
                -H "Accept: application/vnd.github+json" \
                "/repos/$GITHUB_REPOSITORY/actions/runs/$run_id" \
                | jq -r '.pull_requests[0].number')
            
            if [ "$run_pr" = "$pr_number" ]; then
                echo "⏳ Another review is already in progress for PR #$pr_number"
                return 1
            fi
        fi
    done
    
    return 0
}

post_concurrent_message() {
    local pr_number="$1"
    
    local message="⏳ **Fixium Review Already In Progress**

A code review is currently running for this PR. Please wait for it to complete before triggering another review.

You can check the progress in the [Actions tab](https://github.com/$GITHUB_REPOSITORY/actions)."
    
    gh api \
        --method POST \
        -H "Accept: application/vnd.github+json" \
        "/repos/$GITHUB_REPOSITORY/issues/$pr_number/comments" \
        -f body="$message"
}
```

**Note**: GitHub Actions `concurrency` group handles this at workflow level, but this provides user-friendly messaging.

### Task 7: Progress Tracker Script

**Location**: `lib/progress_tracker.sh`

**Purpose**: Post and update progress comments during review

**Functions**:

```bash
#!/bin/bash

# Post initial progress comment
post_progress_comment() {
    local pr_number="$1"
    local filter_info="$2"
    
    local message="🔍 **Fixium Code Review Started**

**Status:** Analyzing PR files...
**Filters:** $filter_info

⏳ This may take a few minutes depending on the size of the PR."
    
    local response=$(gh api \
        --method POST \
        -H "Accept: application/vnd.github+json" \
        "/repos/$GITHUB_REPOSITORY/issues/$pr_number/comments" \
        -f body="$message")
    
    # Extract and return comment ID
    echo "$response" | jq -r '.id'
}

# Update progress comment
update_progress_comment() {
    local comment_id="$1"
    local status="$2"
    local details="$3"
    
    local message="🔍 **Fixium Code Review In Progress**

**Status:** $status
$details"
    
    gh api \
        --method PATCH \
        -H "Accept: application/vnd.github+json" \
        "/repos/$GITHUB_REPOSITORY/issues/comments/$comment_id" \
        -f body="$message"
}

# Finalize progress comment with results
finalize_progress_comment() {
    local comment_id="$1"
    local total_findings="$2"
    local summary="$3"
    local pr_number="$4"
    
    if [ "$total_findings" -eq 0 ]; then
        local message="✅ **Fixium Code Review Complete**

**Result:** No issues found - code looks good! 🎉

Great job maintaining code quality!"
    else
        local message="✅ **Fixium Code Review Complete**

**Findings:** $total_findings issue(s) found

$summary

📝 Review the inline comments below for details and suggestions."
    fi
    
    gh api \
        --method PATCH \
        -H "Accept: application/vnd.github+json" \
        "/repos/$GITHUB_REPOSITORY/issues/comments/$comment_id" \
        -f body="$message"
}

# Post error message
post_error_comment() {
    local comment_id="$1"
    local error_type="$2"
    local error_details="$3"
    
    local message="❌ **Fixium Code Review Failed**

**Error:** $error_type

$error_details

Please try again or contact support if the issue persists."
    
    gh api \
        --method PATCH \
        -H "Accept: application/vnd.github+json" \
        "/repos/$GITHUB_REPOSITORY/issues/comments/$comment_id" \
        -f body="$message"
}
```

**Progress States**:
- 🔍 Review started...
- 📊 Analyzing X files...
- 🔄 Running code analysis...
- ✅ Review complete - Found Y findings
- ❌ Review failed - [error message]

### Task 8: Error Handler Script

**Location**: `lib/error_handler.sh`

**Purpose**: Centralized error handling with user-friendly messages

**Error Types and Messages**:

```bash
#!/bin/bash

handle_error() {
    local error_type="$1"
    local error_details="$2"
    local comment_id="$3"
    
    case "$error_type" in
        "timeout")
            post_error_comment "$comment_id" "Review Timeout" \
                "The review took too long to complete. This usually happens with very large PRs.

**Suggestions:**
- Try reviewing specific files instead of the entire PR
- Break the PR into smaller chunks
- Use severity filters to focus on critical issues first"
            ;;
        
        "no_findings")
            # This is actually success, not an error
            finalize_progress_comment "$comment_id" 0 "" "$PR_NUMBER"
            ;;
        
        "bob_error")
            post_error_comment "$comment_id" "Analysis Error" \
                "The code analysis engine encountered an error.

**Details:**
\`\`\`
$error_details
\`\`\`

**Suggestions:**
- Check if the code compiles successfully
- Verify all dependencies are available
- Try again in a few minutes"
            ;;
        
        "rate_limit")
            post_error_comment "$comment_id" "GitHub API Rate Limit" \
                "GitHub API rate limit has been reached.

**What this means:**
The review completed but couldn't post all comments due to API limits.

**Suggestions:**
- Wait for rate limit to reset (usually within an hour)
- Try again with severity filters to reduce comment count
- Contact administrator to increase rate limits"
            ;;
        
        "invalid_options")
            post_error_comment "$comment_id" "Invalid Filter Options" \
                "The filter options provided are not valid.

**Valid options:**
- \`--severity high,medium,low\`
- \`--type bug,security,maintainability,performance\`
- \`--exclude-severity low\`
- \`--exclude-type maintainability\`

**Example:**
\`Fixium:review --severity high --type bug,security\`"
            ;;
        
        "auth_failed")
            post_error_comment "$comment_id" "Authentication Failed" \
                "Failed to authenticate with the code analysis service.

**This is a configuration issue.**
Please contact the repository administrator."
            ;;
        
        *)
            post_error_comment "$comment_id" "Unknown Error" \
                "An unexpected error occurred during the review.

**Details:**
\`\`\`
$error_details
\`\`\`

Please try again or contact support."
            ;;
    esac
}

# Trap errors and handle them
trap 'handle_error "unknown" "$?" "$PROGRESS_COMMENT_ID"' ERR
```

### Task 10: Main Workflow Script

**Location**: `fixium_workflow.sh`

**Purpose**: Main orchestration script called by Docker entrypoint

**Implementation**:
```bash
#!/bin/bash
set -e

# Source all libraries
source /fixium/lib/github_api.sh
source /fixium/lib/comment_formatter.sh
source /fixium/lib/parse_comment.sh
source /fixium/lib/check_access.sh
source /fixium/lib/progress_tracker.sh
source /fixium/lib/error_handler.sh

# Main workflow
main() {
    local pr_number="$PR_NUMBER"
    local comment_body="$COMMENT_BODY"
    local comment_user="$COMMENT_USER"
    
    # Parse comment
    local parsed=$(parse_fixium_comment "$comment_body")
    local command=$(echo "$parsed" | jq -r '.command')
    
    # Check authorization
    if ! check_user_access "$comment_user" "$FIXIUM_AUTHORIZED_USERS"; then
        post_unauthorized_message "$pr_number" "$comment_user"
        exit 1
    fi
    
    # Post progress comment
    local filter_info=$(build_filter_info "$parsed")
    local comment_id=$(post_progress_comment "$pr_number" "$filter_info")
    export PROGRESS_COMMENT_ID="$comment_id"
    
    # Update progress
    update_progress_comment "$comment_id" "Fetching PR files..." ""
    
    # Run review
    ./review_workflow.sh review-pr "$pr_number" "review_pr${pr_number}.json"
    
    # Update progress
    update_progress_comment "$comment_id" "Submitting review comments..." ""
    
    # Build filter arguments
    local filter_args=$(build_filter_args "$parsed")
    
    # Submit comments with filters
    ./submit_pr_comments.sh "review_pr${pr_number}.json" "$pr_number" $filter_args
    
    # Get summary
    local total_findings=$(jq '.summary.totalFindings // 0' "review_pr${pr_number}.json")
    local summary=$(build_summary "review_pr${pr_number}.json")
    
    # Finalize progress
    finalize_progress_comment "$comment_id" "$total_findings" "$summary" "$pr_number"
}

# Execute main workflow
main "$@"
```
